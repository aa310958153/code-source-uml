一、LSP 系统全景

OpenCode 内置了 36 个 LSP Server 定义，覆盖主流编程语言。系统通过文件扩展名自动匹配并启动对应的 LSP Server，提供语法诊断、跳转定义、引用查找等 IDE 级能力。

设计原则实现方式
零配置36 个内置 Server 定义，按扩展名自动匹配

故障隔离broken Set 记录失败的 server+root 组合，不重复尝试

二、LSPServer.Info 接口定义

每个 LSP Server 都实现相同的 Info 接口：

export interface Info {
    id: string                     // 唯一标识符
    extensions: string[]           // 支持的文件扩展名
    global?: boolean               // 是否全局可用
    root: RootFunction             // (file) => Promise<string | undefined>
    spawn(root: string): Promise<Handle | undefined>
}

interface Handle {
    process: ChildProcessWithoutNullStreams  // LSP 子进程
    initialization?: Record<string, any>     // 初始化参数
}

type RootFunction = (file: string) => Promise<string | undefined>

四、NearestRoot — 项目根目录检测

NearestRoot 是一个工厂函数，根据文件向上查找特定标记文件来确定项目根目录：

const NearestRoot = (includePatterns: string[], excludePatterns?: string[]): RootFunction => {
    return async (file) => {
        // 排除模式：如果找到排除标记，返回 undefined（不启动此 Server）
        if (excludePatterns) {
            const excluded = await Filesystem.up({
                targets: excludePatterns,
                start: path.dirname(file),
                stop: Instance.directory,
            })
            if (excluded.value) return undefined
        }

        // 包含模式：向上查找标记文件
        const files = Filesystem.up({
            targets: includePatterns,
            start: path.dirname(file),
            stop: Instance.directory,
        })
        const first = await files.next()
        if (!first.value) return Instance.directory  // 没找到就用项目根
        return path.dirname(first.value)
    }
}

// 使用示例
export const Typescript: Info = {
    root: NearestRoot(
        ["package-lock.json", "bun.lockb", "bun.lock", "pnpm-lock.yaml", "yarn.lock"],
        ["deno.json", "deno.jsonc"],  // 如果有 deno.json 则不使用 TypeScript Server
    ),
}
Den vs TypeScript 的互斥：TypeScript Server 的 root 函数有排除条件 ["deno.json", "deno.jsonc"]。如果项目中存在 deno.json，TypeScript Server 不会启动，由 Deno Server 接管。这种设计避免了两个 LSP Server 同时处理同一组文件。

五、Server 自动下载与安装

多个 LSP Server 支持自动下载。系统检查本地是否存在，不存在时从 GitHub Releases 或 npm 自动安装：

// ESLint — 从 GitHub 下载 vscode-eslint 源码并编译
export const ESLint: Info = {
    async spawn(root) {
        const serverPath = path.join(Global.Path.bin, "vscode-eslint", "server", "out", "eslintServer.js")
        if (!(await Filesystem.exists(serverPath))) {
            if (Flag.OPENCODE_DISABLE_LSP_DOWNLOAD) return

            const response = await fetch(
                "https://github.com/microsoft/vscode-eslint/archive/refs/heads/main.zip"
            )
            await Filesystem.writeStream(zipPath, response.body)
            await Archive.extractZip(zipPath, Global.Path.bin)
            await $`npm install`.cwd(finalPath)
            await $`npm run compile`.cwd(finalPath)
        }
    },
}

// Pyright — 通过 bun install 安装
export const Pyright: Info = {
    async spawn(root) {
        if (!binary) {
            const js = path.join(Global.Path.bin, "node_modules", "pyright", "dist", "pyright-langserver.js")
            if (!(await Filesystem.exists(js))) {
                if (Flag.OPENCODE_DISABLE_LSP_DOWNLOAD) return
                await Process.spawn([BunProc.which(), "install", "pyright"], {
                    cwd: Global.Path.bin,
                }).exited
            }
        }
    },
}

// Gopls — 通过 go install 安装
if (!bin) {
    if (!which("go")) return
    await Process.spawn(
        ["go", "install", "golang.org/x/tools/gopls@latest"],
        { env: { ...process.env, GOBIN: Global.Path.bin } },
    )
}

六、LSP 状态管理 — state()

LSP 系统通过 Instance.state() 管理生命周期：

const state = Instance.state(
    async () => {
        const clients: LSPClient.Info[] = []
        const servers: Record<string, LSPServer.Info> = {}
        const cfg = await Config.get()

        // LSP 全局禁用
        if (cfg.lsp === false) {
            return { broken: new Set(), servers, clients, spawning: new Map() }
        }

        // 注册所有内置 Server
        for (const server of Object.values(LSPServer)) {
            servers[server.id] = server
        }

        // 实验性过滤（Pyright vs Ty）
        filterExperimentalServers(servers)

        // 用户配置覆盖
        for (const [name, item] of Object.entries(cfg.lsp ?? {})) {
            if (item.disabled) {
                delete servers[name]  // 禁用某个 Server
                continue
            }
            servers[name] = {
                ...existing,
                id: name,
                extensions: item.extensions ?? existing?.extensions ?? [],
                spawn: async (root) => ({
                    process: spawn(item.command[0], item.command.slice(1), {
                        cwd: root,
                        env: { ...process.env, ...item.env },
                    }),
                    initialization: item.initialization,
                }),
            }
        }

        return { broken: new Set(), servers, clients, spawning: new Map() }
    },
    // 销毁回调：关闭所有 LSP Client
    async (state) => {
        await Promise.all(state.clients.map((client) => client.shutdown()))
    },
)

七、getClients — 按需懒启动

LSP Server 不是一开始就全部启动，而是在首次访问对应文件时按需启动：

async function getClients(file: string) {
    const s = await state()
    const extension = path.parse(file).ext || file
    const result: LSPClient.Info[] = []

    for (const server of Object.values(s.servers)) {
        // 1. 扩展名过滤
        if (server.extensions.length && !server.extensions.includes(extension)) continue

        // 2. 确定项目根目录
        const root = await server.root(file)
        if (!root) continue

        // 3. 检查是否已标记为 broken
        if (s.broken.has(root + server.id)) continue

        // 4. 检查是否已有活跃 Client
        const match = s.clients.find((x) => x.root === root && x.serverID === server.id)
        if (match) { result.push(match); continue }

        // 5. 检查是否正在启动中（防重复 spawn）
        const inflight = s.spawning.get(root + server.id)
        if (inflight) {
            const client = await inflight
            if (client) result.push(client)
            continue
        }

        // 6. 首次启动
        const task = schedule(server, root, root + server.id)
        s.spawning.set(root + server.id, task)
        const client = await task
        if (client) {
            result.push(client)
            Bus.publish(Event.Updated, {})  // 通知前端状态变更
        }
    }
    return result
}

schedule() — 启动流程

async function schedule(server, root, key) {
    const handle = await server.spawn(root)
    if (!handle) { s.broken.add(key); return undefined }

    const client = await LSPClient.create({
        serverID: server.id,
        server: handle,
        root,
    })
    if (!client) { handle.process.kill(); return undefined }

    // 防重复：如果恰好已有同 root+server 的 client，杀掉新的
    const existing = s.clients.find(x => x.root === root && x.serverID === server.id)
    if (existing) { handle.process.kill(); return existing }

    s.clients.push(client)
    return client
}

// run vs runAll — 两种调用模式

// run：获取指定文件对应的 Client 执行
async function run<T>(file: string, fn: (client: LSPClient.Info) => Promise<T>) {
    const clients = await getClients(file)
    return Promise.all(clients.map(fn))
}

// runAll：在所有已启动的 Client 上执行
async function runAll<T>(fn: (client: LSPClient.Info) => Promise<T>) {
    const clients = await state().then((x) => x.clients)
    return Promise.all(clients.map(fn))
}

// 使用示例
export async function definition(input) {
    return run(input.file, (client) =>
        client.connection.sendRequest("textDocument/definition", {
            textDocument: { uri: pathToFileURL(input.file).href },
            position: { line: input.line, character: input.character },
        }).catch(() => null),
    ).then((result) => result.flat().filter(Boolean))
}

九、配置覆盖 — 自定义 LSP

用户可以在 opencode.json 中覆盖或新增 LSP Server：

// opencode.json
{
    "lsp": {
        // 禁用内置 Server
        "eslint": { "disabled": true },

        // 覆盖内置 Server 参数
        "pyright": {
            "command": ["pyright-langserver", "--stdio", "--verbose"],
            "env": { "PYTHONPATH": "/custom/path" }
        },

        // 新增自定义 Server
        "my-custom-ls": {
            "command": ["my-ls-binary", "--stdio"],
            "extensions": [".custom"],
            "initialization": { "customSetting": true }
        }
    }
}
自定义 Server 必须声明 extensions：Zod Schema 的 refine 校验要求非内置 Server 必须提供 extensions 数组，否则配置校验失败。
完全禁用 LSP：设置 "lsp": false 可关闭所有 LSP 功能，适用于对性能敏感或不需要诊断的场景。

十、Diagnostics 诊断格式

诊断信息被格式化为易读的字符串：

export namespace Diagnostic {
    export function pretty(diagnostic: LSPClient.Diagnostic) {
        const severityMap = {
            1: "ERROR",
            2: "WARN",
            3: "INFO",
            4: "HINT",
        }
        const severity = severityMap[diagnostic.severity || 1]
        const line = diagnostic.range.start.line + 1    // LSP 是 0-based
        const col = diagnostic.range.start.character + 1

        return `${severity} [${line}:${col}] ${diagnostic.message}`
        // 输出示例: "ERROR [42:10] Type 'string' is not assignable to type 'number'"
    }
}

诊断聚合

export async function diagnostics() {
    const results: Record<string, LSPClient.Diagnostic[]> = {}
    for (const result of await runAll(async (client) => client.diagnostics)) {
        for (const [path, diagnostics] of result.entries()) {
            const arr = results[path] || []
            arr.push(...diagnostics)
            results[path] = arr
        }
    }
    return results
}
// 结果: { "src/app.ts": [diagnostic1, diagnostic2], "src/util.ts": [...] }

多个 LSP Client（如 TypeScript + ESLint）对同一文件的诊断结果被合并在一起。

十一、实验性功能：Ty 替代 Pyright

通过环境变量 OPENCODE_EXPERIMENTAL_LSP_TY 可以切换到 Ruff 团队开发的 Ty 类型检查器：

const filterExperimentalServers = (servers: Record<string, LSPServer.Info>) => {
    if (Flag.OPENCODE_EXPERIMENTAL_LSP_TY) {
        // Ty 启用 → 禁用 Pyright
        if (servers["pyright"]) {
            delete servers["pyright"]
        }
    } else {
        // 默认 → 禁用 Ty
        if (servers["ty"]) {
            delete servers["ty"]
        }
    }
}
