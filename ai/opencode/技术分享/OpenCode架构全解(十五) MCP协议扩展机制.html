一、MCP 是什么？为什么需要它？

Model Context Protocol (MCP) 是一个标准化协议，让 AI 模型能够动态连接外部工具和服务。与内置工具（编译时确定）不同，MCP 工具在运行时发现和注册。

内置工具 vs MCP 工具

维度内置工具MCP 工具
注册时机编译时 / Agent 初始化时运行时动态发现
来源OpenCode 源码外部 MCP Server（本地进程或远程服务）
协议TypeScript 函数JSON-RPC over stdio/SSE/HTTP
示例ReadTool、EditTool、BashToolGitHub API、Jira、数据库查询、自定义工具
权限Agent permission 控制统一的 MCP permission 检查

// packages/opencode/src/mcp/index.ts
if (mcp.type === "remote") {
    // 远程 MCP 服务器 — 尝试两种传输方式
    const transports = [
        { name: "SSE",         transport: new SSEClientTransport(new URL(mcp.url), { authProvider }) },
        { name: "StreamableHTTP", transport: new StreamableHTTPClientTransport(new URL(mcp.url), { authProvider }) },
    ]
    for (const { name, transport } of transports) {
        try {
            const client = new Client({ name: "opencode", version: Installation.VERSION })
            await withTimeout(client.connect(transport), connectTimeout)
            mcpClient = client
            break   // 连接成功，使用此传输方式
        } catch (error) { /* 尝试下一种 */ }
    }
} else {
    // 本地 MCP 服务器 — stdio 传输
    const transport = new StdioClientTransport({
        command: mcp.command[0],
        args: mcp.command.slice(1),
        env: { ...process.env, ...mcp.env },
    })
    await client.connect(transport)
}
协议适用场景传输方式
stdio本地进程（node/python 脚本）stdin/stdout JSON-RPC
SSE远程 HTTP 服务器Server-Sent Events + POST
StreamableHTTP新一代远程服务器双向 HTTP 流
连接回退策略：远程 MCP 先尝试 SSE，失败后尝试 StreamableHTTP。这确保了向后兼容旧版 MCP 服务器，同时支持新版协议。

四、工具转换 — MCP Tool → AI SDK Tool

// mcp/index.ts — convertMcpTool
async function convertMcpTool(mcpTool: MCPToolDef, client: MCPClient, timeout?: number): Promise<Tool> {
    const inputSchema = mcpTool.inputSchema
    const schema: JSONSchema7 = {
        ...(inputSchema as JSONSchema7),
        type: "object",                    // 强制为 object 类型
        properties: (inputSchema.properties ?? {}) as JSONSchema7["properties"],
        additionalProperties: false,
    }

    return dynamicTool({
        description: mcpTool.description ?? "",
        inputSchema: jsonSchema(schema),
        execute: async (args) => {
            return client.callTool(
                { name: mcpTool.name, arguments: (args || {}) as Record<string, unknown> },
                CallToolResultSchema,
                { resetTimeoutOnProgress: true, timeout },
            )
        },
    })
}

转换链路

1
MCP Server 返回工具列表（name + description + JSON Schema）
2
convertMcpTool 将其包装为 AI SDK 的 dynamicTool
3
在 resolveTools() 中，MCP 工具与内置工具合并到同一个 tools 字典
4
AI 模型看到的 MCP 工具和内置工具格式完全一致

五、OAuth 认证流

远程 MCP 服务器默认启用 OAuth 认证：

// OAuth 配置
const oauthDisabled = mcp.oauth === false
const oauthConfig = typeof mcp.oauth === "object" ? mcp.oauth : undefined

const authProvider = new McpOAuthProvider(
    key, mcp.url,
    { clientId: oauthConfig?.clientId, clientSecret: oauthConfig?.clientSecret, scope: oauthConfig?.scope },
    { onRedirect: async (url) => { /* 存储回调 URL */ } },
)

// 连接时如果返回 401，触发 OAuth 流
if (error instanceof UnauthorizedError) {
    status = { status: "needs_auth" }
    pendingOAuthTransports.set(key, transport)
}
OAuth 状态机：connected → needs_auth（需要浏览器授权）→ needs_client_registration（需要注册 client_id）→ connected（授权完成）。

七、资源与 Prompt 暴露

MCP 不仅暴露工具，还暴露资源和 Prompt：

MCP 能力OpenCode 中的映射用途
ToolsAI 可调用的工具数据库查询、API 调用、文件操作
Resources作为文件附件引用配置信息、知识库文档
Prompts注册为 Command预定义的 Prompt 模板
// 获取 MCP 资源
async function fetchResourcesForClient(clientName, client) {
    const resources = await client.listResources()
    for (const resource of resources.resources) {
        const key = sanitizedClient + ":" + sanitizedResource
        commands[key] = { ...resource, client: clientName }
    }
}

清理逻辑

Instance dispose 时，MCP 会：1) 杀死所有子进程树（防止孤儿进程）；2) 关闭所有 MCP Client 连接；3) 清理 pending OAuth transports。

九、与内置工具的集成

在 resolveTools() 中，MCP 工具和内置工具被统一注入到 AI 的工具集中：

// session/prompt.ts — resolveTools
// 1. 内置工具
for (const item of await ToolRegistry.tools(model, agent)) {
    tools[item.id] = wrapWithHooks(item)
}
// 2. MCP 工具
for (const [key, item] of Object.entries(await MCP.tools())) {
    item.execute = wrapWithHooksAndPermission(item)
    tools[key] = item
}
// AI 看到的是一个统一的工具列表，无法区分内置/MCP
权限差异：内置工具的权限由 Agent.permission 精细控制，MCP 工具统一走 permission: key 检查（每个 MCP 工具有独立的权限名）。

十、完整生命周期总结

规律体现
