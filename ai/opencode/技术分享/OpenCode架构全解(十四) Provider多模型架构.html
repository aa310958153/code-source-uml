一、Provider-Agnostic 设计理念

OpenCode 的核心设计目标之一是 Provider-Agnostic — 用户可以使用任何 AI 模型提供商，而无需修改代码。这通过三层抽象实现：

opencode.json
→
Provider.getModel()
→
AI SDK LanguageModel
→
streamText / generateText

核心原则

原则实现
统一接口所有 Provider 都通过 Vercel AI SDK 的 LanguageModelV2 接口调用
模型发现从 models.dev 自动获取每个 Provider 的可用模型列表
自动认证检测环境变量/API Key/OAuth 自动选择认证方式
Custom Loader每个 Provider 可以有特殊的初始化逻辑（如 Bedrock 的区域前缀）
延迟加载Provider SDK 只在首次使用时初始化，避免启动开销

四、Custom Loader — Provider 特殊适配

不同的 Provider 有独特的初始化需求。Custom Loader 提供了每个 Provider 的特殊适配逻辑：

const CUSTOM_LOADERS: Record<string, CustomLoader> = {
    // Anthropic: 注入实验性 header
    async anthropic() {
        return {
            autoload: false,
            options: {
                headers: {
                    "anthropic-beta": "claude-code-20250219,interleaved-thinking-2025-05-14",
                },
            },
        }
    },

    // OpenAI: 使用 Responses API（非 Chat API）
    openai: async () => ({
        autoload: false,
        async getModel(sdk, modelID) { return sdk.responses(modelID) },
    }),

    // GitHub Copilot: GPT-5 用 Responses API，其他用 Chat API
    "github-copilot": async () => ({
        autoload: false,
        async getModel(sdk, modelID) {
            return shouldUseCopilotResponsesApi(modelID)
                ? sdk.responses(modelID)
                : sdk.chat(modelID)
        },
    }),

    // OpenCode: 免费模型自动过滤无 API Key 的付费模型
    async opencode(input) {
        const hasKey = /* 检测认证状态 */
        if (!hasKey) {
            // 删除所有付费模型
            for (const [key, value] of Object.entries(input.models))
                if (value.cost.input !== 0) delete input.models[key]
        }
        return { autoload: Object.keys(input.models).length > 0, options: hasKey ? {} : { apiKey: "public" } }
    },
}
Custom Loader 的三个返回值：
  • autoload: boolean — 是否自动启用（无需用户手动配置）
  • getModel(sdk, modelID) — 自定义模型获取逻辑（如选择 chat vs responses API）
  • options — 传递给 Provider SDK 的额外选项

五、模型发现与自动加载

OpenCode 通过 models.dev 获取所有已知模型的元数据：

模型发现流程

1
获取 Provider 列表 — 从 models.dev 下载所有 Provider 及其模型列表
2
检测认证 — 检查环境变量、Auth 存储、配置文件中的 API Key
3
执行 Custom Loader — 获取 autoload 标志和特殊选项
4
注册可用模型 — autoload=true 的 Provider 自动注册所有模型
5
合并用户配置 — opencode.json 中的 provider/model 配置覆盖默认值
// 模型信息结构
export const Model = z.object({
    id: z.string(),
    name: z.string(),
    providerID: z.string(),
    api: z.object({
        id: z.string(),          // API 中使用的模型 ID
        url: z.string(),         // API 端点
        npm: z.string(),         // 对应的 npm 包名
    }),
    cost: z.object({
        input: z.number(),       // 输入 token 价格（每百万）
        output: z.number(),      // 输出 token 价格
        cache: z.object({ read: z.number(), write: z.number() }).optional(),
        experimentalOver200K: /* 超高 token 数的替代价格 */
    }),
    limit: z.object({
        context: z.number(),     // 上下文窗口大小
        output: z.number(),      // 最大输出 token
    }),
})

六、AWS Bedrock 跨区域推理

Bedrock 是最复杂的 Provider 之一，需要处理跨区域推理前缀：

// Bedrock Custom Loader 中的区域解析
const region = options?.region ?? defaultRegion  // us-east-1
let regionPrefix = region.split("-")[0]          // us

switch (regionPrefix) {
    case "us":
        // Claude、Nova 等需要 us. 前缀
        if (modelID.includes("claude"))
            modelID = `us.${modelID}`              // us.anthropic.claude-sonnet-4-20250514
        break
    case "eu":
        // 欧洲区域特定模型需要 eu. 前缀
        if (region === "eu-west-1" && modelID.includes("claude"))
            modelID = `eu.${modelID}`
        break
    case "ap":
        // 澳大利亚区域使用 au. 前缀
        if (isAustraliaRegion && modelID.includes("claude-sonnet"))
            modelID = `au.${modelID}`
        // 东京区域使用 jp. 前缀
        else if (isTokyoRegion)
            modelID = `jp.${modelID}`
        break
}
认证链优先级：Bearer Token > AWS Profile > Access Key > IAM Role > Web Identity Token > Container Credentials。OpenCode 自动检测可用的认证方式。

七、Google Vertex AI 认证链

"google-vertex": async (provider) => {
    const project = provider.options?.project
        ?? Env.get("GOOGLE_CLOUD_PROJECT")
        ?? Env.get("GCP_PROJECT")
        ?? Env.get("GCLOUD_PROJECT")

    if (!project) return { autoload: false }

    return {
        autoload: true,
        options: {
            project,
            location,
            fetch: async (input, init) => {
                // 使用 google-auth-library 自动获取 access token
                const auth = new GoogleAuth()
                const token = await auth.getAccessToken()
                init.headers = { ...init.headers, Authorization: `Bearer ${token}` }
                return fetch(input, init)
            },
        },
    }
}

八、GitHub Copilot 适配

GitHub Copilot 使用 OpenAI 兼容协议，但有一些特殊处理：

// 模型选择逻辑
async getModel(sdk, modelID) {
    // 如果 SDK 没有 responses API，退回到 languageModel
    if (sdk.responses === undefined && sdk.chat === undefined)
        return sdk.languageModel(modelID)
    // GPT-5+ 使用 Responses API，其他使用 Chat API
    return shouldUseCopilotResponsesApi(modelID)
        ? sdk.responses(modelID)
        : sdk.chat(modelID)
}

function shouldUseCopilotResponsesApi(modelID: string): boolean {
    return isGpt5OrLater(modelID) && !modelID.startsWith("gpt-5-mini")
}

十、成本计算与 Token 统计

// session/index.ts — getUsage
const costInfo = input.model.cost?.experimentalOver200K
    && tokens.input + tokens.cache.read > 200_000
    ? input.model.cost.experimentalOver200K
    : input.model.cost

const cost = new Decimal(0)
    .add(new Decimal(tokens.input).mul(costInfo.input).div(1_000_000))
    .add(new Decimal(tokens.output).mul(costInfo.output).div(1_000_000))
    .add(new Decimal(tokens.cache.read).mul(costInfo.cache?.read ?? 0).div(1_000_000))
    .add(new Decimal(tokens.cache.write).mul(costInfo.cache?.write ?? 0).div(1_000_000))
    .add(new Decimal(tokens.reasoning).mul(costInfo.output ?? 0).div(1_000_000))
    .toNumber()
特殊处理：
  • Anthropic vs 其他：Anthropic 的 inputTokens 不含 cached tokens，而 OpenRouter/OpenAI 包含。代码通过检测 metadata 中的 provider 来修正
  • 200K+ 定价：Anthropic 对超过 200K context 的请求有替代价格（experimentalOver200K）
  • Reasoning tokens：按 output 价格计算（保守估计）
  • Decimal 精度：使用 decimal.js 避免浮点数累积误差

十一、用户自定义 Provider

用户可以在 opencode.json 中配置自定义 Provider 或覆盖内置 Provider：

// opencode.json
{
  "provider": {
    "anthropic": {
      "options": {
        "baseURL": "https://my-proxy.example.com/anthropic"
      }
    },
    "my-local": {
      "npm": "@ai-sdk/openai-compatible",
      "options": {
        "baseURL": "http://localhost:8080/v1",
        "apiKey": "local-key"
      },
      "models": {
        "my-model": {
          "name": "My Local Model",
          "limit": { "context": 32000, "output": 4096 },
          "cost": { "input": 0, "output": 0 }
        }
      }
    }
  }
}

十二、完整生命周期总结

规律体现
AI SDK 统一接口所有 Provider 都通过 LanguageModelV2 调用，streamText/generateText 通用
Custom Loader每个 Provider 的特殊逻辑（API 选择、认证、区域）封装在独立 Loader 中
模型自动发现models.dev + autoload 机制，配置 API Key 后自动启用对应 Provider
延迟初始化Provider SDK 只在首次 getModel 时创建，减少启动开销
Decimal 精度成本计算使用 decimal.js 而非浮点数，避免累积误差
Schema 兼容ProviderTransform 桥接不同 Provider 对 JSON Schema 的差异
环境变量 + Auth + Config三级认证回退：env var → Auth store → opencode.json options
